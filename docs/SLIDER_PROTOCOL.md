# 💡 학습 슬라이더 생성 프로토콜 (Slider Generation Protocol)

본 문서는 정보처리기능사 및 기타 자격증 학습을 위한 AI 요약 슬라이더 생성 시 반드시 준수해야 할 규칙을 정의합니다.

## 🎨 시각적 규칙 (Visual Rules)
1. **실사 이미지 중심**: 이모지나 단순 이모티콘 대신, 개념을 직관적으로 설명할 수 있는 **실사 풍의 생성형 이미지**를 사용합니다.
2. **비유(Metaphor) 활용**: 추상적인 IT 개념(DB, 네트워크 등)을 일상적인 사물이나 상황(도서관, 우체국, 요리 등)에 비유하여 시각화합니다.
3. **텍스트 무결성**: 이미지 내에 포함되는 글자가 깨지거나 왜곡되지 않도록 주의하며, 가급적 중요한 설명은 이미지 외부 텍스트 영역에 배치합니다.

## ✍️ 내용 및 수준 규칙 (Content Rules)
1. **대상 수준**: 초등학생이나 중학생이 읽어도 단번에 이해될 정도로 **최대한 쉽고 친절하게** 설명합니다.
2. **금기 사항**: 본문 어디에도 "초등학생 수준", "중학생용"이라는 단어를 **절대 직접적으로 언급하지 않습니다.** (설명 방식만 쉽게 유지)
3. **언어**: 기본적으로 **한글**을 사용하며, 전문 용어는 병기하거나 비유를 통해 먼저 설명합니다.
4. **문제 풀이 전략**: 단순히 개념 설명에서 끝내지 않고, 해당 개념이 **실제 시험 문제에서 어떻게 활용되는지(풀이 팁)**를 반드시 포함합니다.

## 🏗️ 구조적 규칙 (Structure Rules)
1. **10장 내외 구성**: 모든 세트는 기본적으로 **10장 내외**의 슬라이드로 구성하며, 개념의 복잡도에 따라 필요시 추가할 수 있습니다.
2. **레이아웃 참조**: `승강기기능사 1세트`의 배치 방식을 표준으로 삼습니다.
3. **데이터 연동**: 해당 세트에 포함된 실제 기출문제의 키워드를 추출하여 반영합니다.

## 🔑 운영 규칙 (Operational Rules)
1. **키 로테이션**: 제공된 4개의 Gemini API 키를 로테이션하여 사용량 제한(Quota)을 효율적으로 관리합니다.
2. **영속성**: 생성된 데이터는 `src/summaries/[과목명]/[단원]_[세트]세트.json` 경로에 저장하여 재사용합니다.
